\# SC-200 Day 08 Summary



\## Day Focus

Microsoft Sentinel Automation and Response



\## What I Learned

Today I reviewed how Microsoft Sentinel supports automation through automation rules and playbooks. I learned that automation rules can run when incidents are created or updated, and that playbooks use Azure Logic Apps to perform automated response workflows.



\## Key Concepts Covered

\- Automation Rules

\- Playbooks

\- Azure Logic Apps

\- Incident-triggered automation

\- Response orchestration



\## Lab Outcome

The lab showed how Sentinel can reduce manual analyst work by automating common incident response actions. This improves SOC efficiency and helps standardize the response process.



\## Reflection

This day helped reinforce the importance of automation in a SOC. It showed how security teams can respond faster and more consistently by using built-in Sentinel automation features.
