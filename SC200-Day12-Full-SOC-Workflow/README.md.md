\# SC-200 Day 12 — Full SOC Workflow Summary



\## Overview



This capstone lab demonstrates a complete Security Operations Center (SOC) workflow using Microsoft Sentinel and Microsoft Defender XDR.



The objective was to bring together detection, investigation, validation, and response into a structured and repeatable security operations process aligned with real-world SOC practices.



\---



\## What Was Completed



\- Reviewed Microsoft Sentinel environment and configuration

\- Investigated incidents and alerts

\- Analyzed entities using the investigation graph

\- Performed KQL queries to validate security events

\- Reviewed automation rules and playbooks

\- Validated Microsoft Defender XDR integration (no active alerts observed)

\- Documented architecture and SOC workflow

\- Created full business case and SOP documentation

\- Uploaded all artifacts to GitHub

\- Updated Azure DevOps and Excel tracker



\---



\## Key Learnings



\- Microsoft Sentinel acts as the central SOC platform for monitoring and investigation

\- Microsoft Defender XDR provides security signals that can be correlated in Sentinel

\- KQL is essential for validating and investigating security data

\- Automation improves SOC efficiency and reduces manual workload

\- A structured SOC workflow improves consistency and response capability



\---



\## AI Integration



AI-assisted investigation was simulated based on Microsoft Security Copilot capabilities.



The AI workflow was used to:

\- summarize incident context

\- identify suspicious activity

\- suggest next investigation steps



All AI outputs were validated manually to ensure accuracy.



\---



\## Outcome



This lab demonstrates a full end-to-end SOC workflow:



Detection → Investigation → Validation → Response



Even with limited live data, the complete process and tool integration were successfully reviewed and documented.



\---



\## Final Reflection



This Day 12 lab completes the SC-200 hands-on journey and represents a real-world SOC operations model.



It highlights the importance of combining tools, processes, and structured workflows to build an effective security operations capability.



This project is portfolio-ready and reflects practical experience in Microsoft security operations.
