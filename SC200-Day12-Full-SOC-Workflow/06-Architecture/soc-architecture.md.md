\# SC-200 Day 12 Architecture



\## Architecture Overview



This lab demonstrates a full SOC workflow using Microsoft Sentinel and Microsoft Defender XDR.



The architecture shows how security data is collected, analyzed, investigated, and used to support response actions.



\## Core Components



\- Users

\- Devices

\- Microsoft Defender XDR

\- Microsoft Sentinel

\- Log Analytics Workspace

\- Analytics Rules

\- Incidents and Alerts

\- Investigation Graph

\- KQL Queries

\- Automation Rules

\- Playbooks / Logic Apps



\## Data Flow



Users / Devices / Identities

↓

Microsoft Defender XDR

↓

Microsoft Sentinel

↓

Analytics Rules

↓

Incidents and Alerts

↓

Investigation and KQL Validation

↓

Automation / Playbooks

↓

SOC Response



\## What This Diagram Proves



This architecture proves that Microsoft Sentinel can act as the central SOC platform for monitoring, detection, investigation, and response.



It also shows how Microsoft Defender XDR can support Sentinel by contributing security signals for centralized analysis.



\## Security Value



\- Centralized visibility

\- Faster detection

\- Structured investigation

\- Improved response workflow

\- Better SOC operational maturity
