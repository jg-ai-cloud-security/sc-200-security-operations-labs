\# Threat Hunting Overview



\## What is Threat Hunting?

Threat hunting is the proactive process of searching through security data to identify suspicious behaviour that may not yet have triggered a confirmed alert or incident.



\## Why Threat Hunting Matters

Not all threats are detected immediately by analytics rules or automated detections. Some attacks begin as weak signals, such as repeated failed sign-ins, unusual account activity, or suspicious trends across multiple data sources.



\## Threat Hunting in Microsoft Sentinel

Microsoft Sentinel supports threat hunting by allowing analysts to run built-in and custom KQL queries against collected logs. This helps analysts investigate suspicious activity directly in the data.



\## Why KQL is Important

Kusto Query Language (KQL) is used to search, filter, summarise, and analyse log data in Microsoft Sentinel. It is an essential skill for identifying suspicious behaviour and validating security findings.



\## Alert-Based Investigation vs Threat Hunting

Alert-based investigation begins when the system generates an alert or incident.

Threat hunting begins when the analyst proactively searches for suspicious patterns in the data, even if no alert has been triggered yet.
