\# SC-200 Day 05 — Incident Investigation



\## Investigation Summary



A scheduled analytics rule was configured to detect brute force login attempts by monitoring failed authentication events.



The detection logic identified repeated login failures associated with a specific account within a five-minute time window.



\## Investigation Steps



1\. Review the incident created by Microsoft Sentinel

2\. Analyze related alerts

3\. Review associated entities such as users and computers

4\. Examine event logs for patterns indicating credential attack attempts



\## Findings



Multiple failed authentication attempts were detected which may indicate a brute force attack or misconfigured authentication attempts.



\## SOC Response



Recommended actions include:



• Investigate the affected account  

• Check for password compromise  

• Implement account lockout policies  

• Monitor future login activity


# SC-200 Day 05 — Investigation Result

## Investigation Summary

A KQL query was executed in Microsoft Sentinel to detect repeated failed login attempts that may indicate a brute force attack.

The query analyzes Windows Security Event logs with EventID 4625 (failed logon attempts).

---


## Log Ingestion Validation

Security logs were successfully ingested into Microsoft Sentinel from the Windows host SC200-WIN-SEC01.

The SecurityEvent table shows Windows Security auditing events generated by the Microsoft-Windows-Security-Auditing provider.

This confirms that the data ingestion pipeline from the Windows host to the Log Analytics workspace and Microsoft Sentinel is functioning correctly.



## Query Used

```kql
SecurityEvent
| where EventID == 4625
| summarize FailedAttempts = count() by Account, Computer, bin(TimeGenerated, 5m)
| where FailedAttempts > 5
