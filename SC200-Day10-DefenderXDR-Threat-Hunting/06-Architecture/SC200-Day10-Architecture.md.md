\# SC-200 Day 10 Architecture Notes



\## Title

Microsoft Defender XDR Threat Hunting Workflow



\## What This Diagram Should Show

\- Security telemetry from endpoints, identities, email, and alerts

\- Microsoft Defender XDR as the unified portal

\- Advanced Hunting as the analyst investigation layer

\- KQL queries used by the SOC analyst

\- Hunting results feeding investigation and response decisions



\## What This Diagram Proves

This architecture proves that Microsoft Defender XDR centralizes security telemetry and enables proactive threat hunting through Advanced Hunting and KQL. It shows how analysts move from raw telemetry to investigation and SOC decision-making.



\## Final Reflection

This lab helped reinforce that proactive threat hunting is a core SOC capability. Instead of waiting only for alerts, analysts can use Defender XDR to query data directly and identify suspicious activity across the environment.
