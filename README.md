# SC-200 Security Operations Analyst — Hands-On Labs

This repository documents my hands-on learning journey for the Microsoft SC-200 Security Operations Analyst certification.

The labs demonstrate practical SOC operations using Microsoft Sentinel, including log ingestion, detection engineering, incident investigation, and threat hunting.

## Skills Demonstrated

• Security monitoring with Microsoft Sentinel  
• Detection engineering using KQL queries  
• Incident investigation workflows  
• Automation and SOAR playbooks  
• Threat hunting techniques  

## Lab Structure

Each lab day contains the following documentation:

01-Concepts — Key security concepts  
02-Lab-Steps — Step-by-step lab instructions  
03-Evidence — Screenshots of implementation  
04-KQL-Queries — Detection and hunting queries  
05-Investigation — SOC investigation notes  
06-Architecture — Security architecture diagrams  
07-Azure-DevOps — Sprint tracking and documentation

## Lab Progress

Day 01 — Sentinel environment setup  
Day 02 — Microsoft Defender XDR investigation  
Day 03 — Log ingestion with Azure Arc  
Day 04 — Sentinel detections + SOAR automation  
Day 05 — Detection engineering (analytics rules)  
Day 06 — Threat hunting with KQL

\## Cost Hygiene


To avoid unnecessary charges:

\- Start small and expand ingestion sources gradually

\- Use budgets + alerts

\- Disable connectors you’re not testing

\- Remove test rules/playbooks when done

\- Delete lab resources after evidence capture (if not needed)
