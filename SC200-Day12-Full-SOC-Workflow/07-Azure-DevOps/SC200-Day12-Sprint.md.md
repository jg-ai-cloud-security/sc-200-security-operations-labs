\# Task Descriptions — SC-200 Day 12



\---



\## Task 1 — Review Sentinel Environment

Review the Microsoft Sentinel workspace to understand the current security monitoring setup, including data connectors, analytics rules, and overall configuration.



\---



\## Task 2 — Investigate Incidents and Alerts

Analyze security incidents and associated alerts to understand detected threats, severity levels, and how alerts are grouped into incidents.



\---



\## Task 3 — Analyze Entities Using Investigation Graph

Use the investigation graph to explore relationships between users, devices, IP addresses, and other entities involved in the incident.



\---



\## Task 4 — Perform KQL Log Queries

Execute KQL queries against relevant tables (e.g., SigninLogs, SecurityEvent) to validate suspicious activity and gather additional evidence.



\---



\## Task 5 — Review Automation Rules and Playbooks

Examine available automation rules and playbooks to understand how incident response actions can be triggered automatically.



\---



\## Task 6 — Validate Defender XDR Integration

Confirm integration between Microsoft Sentinel and Microsoft Defender XDR by reviewing alerts, incidents, and entity correlation across both platforms.



\---



\## Task 7 — Document Findings and Architecture

Document investigation findings, observations, and create an architecture diagram representing the SOC workflow and data flow.



\---



\## Task 8 — Update GitHub Repository

Organize and upload all lab artifacts, including documentation, screenshots, KQL queries, and architecture diagrams to the GitHub repository.



\---



\## Task 9 — Update Excel Tracker

Update the study and progress tracker with completed tasks, outcomes, and status for SC-200 Day 12.



\---



\## Task 10 — Publish LinkedIn Project

Create and publish a LinkedIn project post summarizing the Day 12 lab, highlighting SOC workflow, investigation techniques, and tools used.





\---



\## Task 11 — Simulate AI-Assisted Investigation

Simulate the use of AI (Security Copilot) to summarize incidents, identify suspicious activity, and suggest investigation steps, then validate results using Sentinel data.
